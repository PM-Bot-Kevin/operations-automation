# 工作区维护规则

## 维护主口径

- 后续新增规则，优先改仓库文档和脚本，不要只留在聊天里
- 发布、回滚、GitHub 备份、跨工作区协作，统一以仓库内脚本和配置为准
- 当前工作区如果依赖其他工作区，只能连接对方正式入口
- 涉及千帆后台的流程，默认必须保持只读、低频、最小操作；如果要查订单信息，优先按单店分批执行，并保持不规则停顿，不允许固定节奏高频查询

## 正式入口约束

允许：

- 已承诺稳定的 CLI 脚本
- 固定服务别名
- 固定 HTTP 地址
- 正式环境 `current` 入口

禁止：

- 仓库镜像
- 临时副本
- worktree
- 手工复制目录
- 历史目录
- 聊天里提到的随手路径

## 维护人需要记住的事

- 正式发布前自动先跑治理校验
- GitHub 是代码备份，不是真实业务数据备份
- 回滚只切代码，不碰 `runtime/`
- 当前长期只保留 `main`
