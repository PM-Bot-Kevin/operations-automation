# 千帆后台极度保守执行规则

## 一句话口径

凡是涉及小红书千帆后台的需求，默认目标不是“尽快做完”，而是“尽量像真人、尽量少操作、尽量不触发风控”。

## 默认执行方式

1. 先在飞书或别的外部表里把目标订单缩小到最少。
2. 一次只处理一个店铺，不混跑。
3. 一个批次只查少量订单，默认每轮不超过 5 单。
4. 每轮结束后必须停一下，再决定是否继续下一轮。
5. 全程优先复用当前已打开的页面状态，不刷新、不乱跳、不多开。
6. 需要重新打开千帆页面时，优先复用用户本机现有 Chrome 资料。
7. 需要读页面控件时，优先走 macOS 原生无障碍树做轻量读取，不走高频整页扫描，也不默认依赖远程调试桥。
8. 遇到活动蒙层、公告弹窗这类阻断层时，只允许优先点“关闭 / 取消 / 稍后 / 跳过”这类安全控件清理页面，不点“去参与 / 去开通 / 去设置 / 确认提交”这类正向按钮。

## 明确禁止

- 固定节奏连续查询
- 并发查询多个订单或多个店铺
- 为了继续执行而频繁刷新页面
- 为了提速直接调用千帆后台接口
- 未经允许在千帆后台做任何写操作

## 发现这些情况要立刻停

- 出现验证码、重新登录、权限异常
- 页面明显变慢、报错、结构异常
- 需要反复点、反复刷才能继续
- 查询结果不稳定，无法确认规格或状态

## 适用范围

- 查订单 SKU / 规格
- 查签收状态
- 查订单状态
- 其他只读查询类千帆需求

## 维护要求

- 以后只要新增千帆相关需求，默认先复用本文件和 `config/xhs_qianfan_guardrails.json`
- 如果后续经验有变化，先改本文件和配置，再开始执行
